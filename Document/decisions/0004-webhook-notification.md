# ADR-0004 第一版通知使用 Webhook / HTTP POST

## 状态

已接受

## 背景

项目需要通知能力，但不同用户可能使用企业微信、钉钉、飞书、自建服务或其他平台。直接绑定某个平台会增加集成和维护成本。

## 决策

第一版通知能力使用通用 Webhook / HTTP POST。

## 理由

- 适配范围广。
- 实现成本低。
- 可以通过模板覆盖多数机器人通知需求。
- 后续可在 Webhook 基础上封装特定平台预设。

## 后果

- 用户需要自行配置 URL、Header 和 Body 模板。
- 需要提供测试发送和模板变量预览。
- 敏感字段需要脱敏和安全存储。
