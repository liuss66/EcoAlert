# ADR-0002 第一版使用本地 JSON 文件存储

## 状态

已接受

## 背景

第一版数据规模较小，主要包括视频源、分组、状态历史、算法配置和通知配置。项目需要快速落地，不希望过早引入数据库迁移和查询层复杂度。

## 决策

第一版使用本地 JSON 文件存储配置和历史数据。

## 理由

- 实现简单，可直接检查和备份。
- 适合单机单管理员场景。
- 与“不做云服务”的产品边界一致。
- 后续可通过 `schema_version` 做配置迁移。

## 后果

- 状态历史需要截断，避免文件无限增长。
- 并发写入需要集中在后端状态层处理。
- 如果后续查询复杂度提高，应迁移到 SQLite。

## 迁移触发条件

- 状态历史超过数万条并影响查询性能。
- 需要复杂筛选、分页、统计。
- 需要事务一致性或多表关联。
