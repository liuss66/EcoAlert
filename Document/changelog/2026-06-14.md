# 2026-06-14 变更日志

## 算法与报警

- 人员检测策略改为固定办公室摄像头场景下的 ROI 运动代理。
- 运动检测适配人员 ROI，ROI 外运动不计入人员判断。
- 增强运动检测抗干扰：
  - 降低微动漏检风险。
  - 过滤小面积紧凑设备闪烁。
  - 降低帧间噪声影响。
- 人员存在状态加入 5 分钟保持：
  - 常规模型任一帧检测到有人后，后续 5 分钟保持有人。
  - VLM 检测到人后，同样保持 5 分钟。
- VLM 兜底策略更新：
  - 常规模型 5 分钟没有发现人后才触发 VLM。
  - VLM 连续两次确认无人后，且灯亮，才触发报警和通知。
  - 支持 MiniMax-M3 多模态请求格式，使用 `thinking.disabled`、`max_completion_tokens` 和 `image_url.detail`。
- 报警图标增加双层确认进度提示：外层粉色展示 VLM 无人确认进度，内层红色展示两次 VLM 无人后的报警倒计时。

## 前端与诊断

- 实时卡片 `state-icons` 改为分别显示：
  - VLM 人员判断小人。
  - 常规模型人员判断小人。
  - 灯光状态。
  - 报警状态。
- VLM 人员判断默认值为 0/无人，并放在常规模型小人左侧。
- 开发者模式新增“检测历史”菜单，支持按视频源查看灯光、人员、报警和算法参数曲线。
- 基础设置中“调试”从“数据存储”面板移出，独立为设置 tab。
- 关于页移除开源协议相关展示文案。

## 通知

- 开发者模式开启时自动补充默认企业内部通知渠道，方便联调通知发送。
- 通知表单新增默认 Body 模板。
- 飞书 / 企业微信 / QQ 的 API 凭证模式补齐字段说明：
  - 飞书：群 `chat_id`。
  - 企业微信：`AgentID` 和接收人 `touser`。
  - QQ：普通聊天机器人配置只需要 `App ID / Client Secret`，不再强制要求群 `group_openid`。
- 修复飞书 API 模式 token 使用顺序，发送群消息优先使用 `tenant_access_token`。
- 通知目标列表在 API 模式下显示接收目标，避免地址列为空。
- 后端通知目标校验支持平台 API 凭证模式，不再强制要求 Webhook URL。
- 修复飞书自定义机器人 Webhook 模式：默认使用 `msg_type=text` 和 `content.text` 格式；默认模板不再覆盖后端内置飞书报文渲染。
- 优化通知配置按钮布局，将扫码绑定、验证凭证、发送测试、保存操作分组展示。
- 通知配置页改为“新增通知”按钮展开表单，列表默认优先展示；保存后自动收起，编辑时再展开。
- 修复通知保存被浏览器隐藏字段校验拦截的问题：Webhook URL 和 API 凭证字段按当前渠道/模式动态启用必填。
- 默认通知模板和后端文本通知统一使用“视频源 / 区域”字段，明确告警来自哪个区域和哪个视频源。
- 通知 payload 新增 `source_url` 模板变量，通用 Webhook / 自建服务可据此按视频源抓图；飞书、企业微信、QQ 文本机器人默认发送文字，图片需后续接入各平台媒体上传后再启用。

## MATLAB
