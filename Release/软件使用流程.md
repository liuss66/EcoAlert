# EcoAlert 软件使用流程

> 版本：v1.0  
> 日期：2026-06-14  
> 发布包：EcoAlert 1.0.0

## 1. 快速开始

1. 运行 `EcoAlert_1.0.0_x64-setup.exe` 安装程序；如需 MSI，可运行 `EcoAlert_1.0.0_x64_en-US.msi`。
2. 启动 EcoAlert。
3. 使用默认密码 `admin123` 登录。
4. 进入「基础设置」修改管理员密码。
5. 进入「视频管理」或「实时监控」添加视频源。
6. 进入「检测配置」设置算法时段、报警保持时间和 ROI。
7. 进入「通知管理」配置报警通知目标并发送测试。
8. 回到「实时监控」进行日常值守。

## 2. 测试视频推流

如果要使用仓库内测试视频，需要先启动本地 HLS 推流器：

```bash
cd Tools
pip install -r requirements.txt
python -m push_streamer.cli --config config.example.yaml
```

默认视频地址：

```text
http://127.0.0.1:8080/cam-1/index.m3u8
http://127.0.0.1:8080/cam-2/index.m3u8
```

## 3. 核心流程

### 3.1 登录与账号

- 默认密码：`admin123`。
- 登录后进入「基础设置」修改密码。
- 登录态随应用重启失效，重启后需重新登录。

### 3.2 视频源管理

- 支持 HLS、MP4、摄像头、RTSP 四类视频源。
- 推荐生产优先接入 HLS。
- 视频源可新增、编辑、删除、启用、停用。
- 可填写位置 / 备注，便于报警通知定位。

### 3.3 分组管理

- 可按楼层、区域或业务场景创建分组。
- 视频卡可拖拽到其他分组。
- 分组支持折叠、重命名和删除。
- 删除分组时，组内通道回到默认分组。
- 分组标题栏有「域检测」开关。开启后，同组所有启用视频源都报警时才发送外部报警通知；任一路恢复后发送一次恢复通知。

### 3.3.1 域检测建议

当一个办公室由多路摄像头覆盖，且灯光会互相反光或干扰时，建议把这些视频源放到同一分组并开启「域检测」。这样单路画面的误判仍会在页面中显示，方便排查，但不会立即外发通知；只有整组都判断为无人且存在开灯风险时才通知。

### 3.4 实时监控

- 视频卡展示画面、通道名称、位置、在线状态和算法状态。
- 人员图标表示检测到有人。
- 灯光图标表示检测到开灯。
- 报警图标表示进入报警态。
- 报警 Banner 会列出当前异常通道。

### 3.5 监控总览

- 查看视频源总数、在线数量、总码率、当前有人、报警中。
- 查看各通道状态。
- 查看最近状态变化。
- 对报警记录执行确认和恢复处理。

### 3.6 检测配置

- 配置算法是否启用、启用星期、起止时间。
- 配置简单模型执行周期。
- 配置人员运动阈值、报警保持时间、恢复保持时间。
- 支持全局配置和通道级独立配置。

### 3.7 ROI 标定

1. 进入「检测配置」-「ROI 标定」。
2. 选择视频源。
3. 拖动预览框选择检测区域。
4. 调整开灯色彩阈值和关灯色彩阈值。
5. 点击「测试 ROI」查看检测结果。
6. 点击「保存 ROI」。

### 3.8 通知管理

- 支持通用 Webhook、飞书、企业微信、QQ。
- 支持报警触发、报警恢复或全部事件。
- 支持冷却时间，避免重复通知。
- 支持测试发送、通知历史和失败重发。

## 4. 推荐上线检查清单

- 已安装 WebView2。
- `ffmpeg -version` 可正常执行。
- 所有视频源在 EcoAlert 中可播放。
- 已修改默认密码。
- 灯光互相干扰的区域已开启「域检测」。
- 关键通道已完成 ROI 标定。
- 开灯 / 关灯 / 有人 / 无人场景均完成测试。
- 算法启用时段符合业务规则。
- 通知目标测试发送成功。
- 报警保持时间和恢复策略符合现场要求。

## 5. 常见问题

### HLS 视频无法播放

检查 HLS URL、推流器 / 转码服务、防火墙、`.ts` 分片和本机 `127.0.0.1` 访问权限。

### 报警过于敏感

缩小 ROI、提高人员运动阈值、延长报警保持时间，或查看检测历史中的 `colorScore` / `motionScore` 曲线。

### 报警不触发

检查算法是否启用、当前时间是否在启用时段、视频是否在线、ROI 是否正确覆盖目标区域、报警保持时间是否达到。

## 6. 当前版本限制

- 人员检测当前仍是 ROI 内运动代理，不等同于生产级人形检测。
- 真实 VLM provider 尚待接入。
- RTSP 需要后续转码链路完善。
- 当前不提供录像存储、多用户权限和时间轴回放。
